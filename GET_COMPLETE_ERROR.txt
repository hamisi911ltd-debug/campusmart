Based on the pattern, here are the most common causes:

ISSUE 1: DATABASE_URL is not set in campusmart service
────────────────────────────────────────────────────────────────
Symptom: Error message is vague or missing
Fix:
  1. Go to campusmart service (NOT Postgres)
  2. Variables tab
  3. Add DATABASE_URL = ${{Postgres.DATABASE_PRIVATE_URL}}
  4. Save

ISSUE 2: DATABASE_URL is empty
────────────────────────────────────────────────────────────────
Symptom: "undefined" or connection errors
Fix:
  1. Set DATABASE_URL = ${{Postgres.DATABASE_PRIVATE_URL}}
  2. Make sure Postgres service is running

ISSUE 3: Using wrong service reference
────────────────────────────────────────────────────────────────
Symptom: "Cannot resolve reference" errors
Fix:
  1. Make sure both services in same project
  2. Postgres service must be named "Postgres"
  3. If different name, adjust reference

ISSUE 4: Postgres service not running
────────────────────────────────────────────────────────────────
Symptom: Connection refused or timeout
Fix:
  1. Check Postgres service status
  2. Should show "Active" or "Running"
  3. If not, restart it
