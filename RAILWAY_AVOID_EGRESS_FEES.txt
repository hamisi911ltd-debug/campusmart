⚠️ WARNING: Using DATABASE_PUBLIC_URL costs money!

Railway charges "egress fees" when your backend connects to the
database through the public internet. This can add up quickly!

═══════════════════════════════════════════════════════════════
  💸 WHAT ARE EGRESS FEES?
═══════════════════════════════════════════════════════════════

EGRESS = Data leaving Railway's network to the public internet

When you use DATABASE_PUBLIC_URL:
  Backend → Public Internet → Railway Edge → Database
  ↑                                           ↑
  Egress fee charged here    Ingress fee charged here

Cost: ~$0.10 per GB of data transferred

For a busy app:
  • 1000 database queries/day
  • ~10KB per query
  • = ~10MB/day = ~300MB/month
  • = ~$0.03/month (small)

But for heavy usage:
  • 100,000 queries/day
  • = 1GB/day = 30GB/month
  • = ~$3/month (adds up!)

═══════════════════════════════════════════════════════════════
  ✅ SOLUTION: USE PRIVATE NETWORKING (FREE!)
═══════════════════════════════════════════════════════════════

When you use DATABASE_PRIVATE_URL:
  Backend → Railway Private Network → Database
  ↑                                    ↑
  NO fees!              NO fees!

Cost: $0 (included in Railway plan)

Benefits:
✅ No egress fees
✅ Faster connections
✅ More secure
✅ No SSL configuration needed

═══════════════════════════════════════════════════════════════
  🚀 HOW TO USE PRIVATE URL (2 MINUTES)
═══════════════════════════════════════════════════════════════

STEP 1: Go to Railway Dashboard
────────────────────────────────────────────────────────────────
https://railway.app/dashboard

STEP 2: Update DATABASE_URL
────────────────────────────────────────────────────────────────
1. Click: campusmart project
2. Click: campusmart service (backend)
3. Click: Variables tab
4. Find: DATABASE_URL
5. Edit to: ${{Postgres.DATABASE_PRIVATE_URL}}
6. Save

STEP 3: Remove SSL Variables (Not needed anymore)
────────────────────────────────────────────────────────────────
1. Delete: PGSSLMODE (if exists)
2. Delete: NODE_TLS_REJECT_UNAUTHORIZED (if exists)
3. Save

STEP 4: Wait and Test
────────────────────────────────────────────────────────────────
1. Wait 2 minutes for redeploy
2. Check logs for "Database schema verified"
3. Test app

Done! No more egress fees! ✅

═══════════════════════════════════════════════════════════════
  💡 UNDERSTANDING THE WARNING
═══════════════════════════════════════════════════════════════

Railway's warning message:
"This variable references a public endpoint through this variable:
DATABASE_PUBLIC_URL -> Postgres.DATABASE_PUBLIC_URL -> 
Postgres.RAILWAY_TCP_PROXY_DOMAIN

Connecting to a public endpoint will incur egress fees."

What this means:
• DATABASE_PUBLIC_URL uses public internet
• Goes through TCP proxy (public endpoint)
• Every database query costs money (egress)
• Can add up to significant costs

Solution:
• Use DATABASE_PRIVATE_URL instead
• Uses Railway's internal network
• No egress fees
• Faster and more secure

═══════════════════════════════════════════════════════════════
  🔧 WHAT IF DATABASE_PRIVATE_URL DOESN'T EXIST?
═══════════════════════════════════════════════════════════════

If you don't see DATABASE_PRIVATE_URL in Postgres variables:

OPTION 1: Check Different Name
────────────────────────────────────────────────────────────────
Look for:
  • DATABASE_URL_INTERNAL
  • RAILWAY_PRIVATE_DOMAIN
  • Any URL with .railway.internal

OPTION 2: Wait for Provisioning
────────────────────────────────────────────────────────────────
If Postgres was just created:
  • Wait 2-3 minutes
  • Refresh Variables tab
  • Should appear

OPTION 3: Use DATABASE_URL (Check if Private)
────────────────────────────────────────────────────────────────
1. Click "Show" on DATABASE_URL value
2. If it has .railway.internal → It's private! ✅
3. If it has .railway.app → It's public ❌

OPTION 4: Contact Railway Support
────────────────────────────────────────────────────────────────
If private URL truly doesn't exist:
  • Your plan might not support it
  • Contact Railway support
  • Or upgrade plan

OPTION 5: Accept Egress Fees (Not Recommended)
────────────────────────────────────────────────────────────────
If you must use public URL:
  • Use DATABASE_URL or DATABASE_PUBLIC_URL
  • Add: PGSSLMODE=no-verify
  • Accept egress fees
  • Monitor costs in Railway dashboard
