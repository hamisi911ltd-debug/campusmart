WHAT IS PRIVATE NETWORKING?
Railway's internal network that connects services without going
through the internet. No SSL needed because it's already secure.

BENEFITS:
✅ No SSL configuration required
✅ Faster database connections
✅ More secure (not exposed to internet)
✅ No certificate issues
✅ Works immediately

═══════════════════════════════════════════════════════════════
  STEP 1: CHECK IF PRIVATE NETWORKING IS AVAILABLE
═══════════════════════════════════════════════════════════════

1. Go to: https://railway.app/dashboard

2. Click your campusmart project

3. You should see TWO services:
   • campusmart (your backend)
   • Postgres (your database)

4. Both must be in the SAME project for private networking

✅ If you see both → Continue to Step 2
❌ If not → Use alternative methods

═══════════════════════════════════════════════════════════════
  STEP 2: GET PRIVATE NETWORK CONNECTION URL
═══════════════════════════════════════════════════════════════

1. Click on "Postgres" service (the database)

2. Click "Connect" tab

3. Look for "Private Network" or "Internal URL" section

4. You should see a URL like:
   postgresql://postgres:password@postgres.railway.internal:5432/railway

   Key difference: ".railway.internal" instead of ".railway.app"

5. Copy this entire URL

6. If you don't see "Private Network" section:
   • Check if both services are in same project
   • Railway might call it "Internal URL" or "Service URL"
   • Look for any URL with ".railway.internal"

═══════════════════════════════════════════════════════════════
  STEP 3: UPDATE BACKEND TO USE PRIVATE NETWORK
═══════════════════════════════════════════════════════════════

1. Go to campusmart service (your backend)

2. Click "Variables" tab

3. Find DATABASE_URL variable

4. Click "Edit" or the pencil icon

5. Replace the value with private network URL from Step 2

   OLD (public):
   postgresql://postgres:xxx@containers-us-west-xxx.railway.app:5432/railway

   NEW (private):
   postgresql://postgres:xxx@postgres.railway.internal:5432/railway

6. Click "Save" or "Update"

7. Railway will automatically redeploy (wait 2 minutes)

═══════════════════════════════════════════════════════════════
  STEP 4: VERIFY IT'S WORKING
═══════════════════════════════════════════════════════════════

1. After redeploy completes, click "View Logs"

2. Look for these messages:
   ✅ "Database: Initialized PostgreSQL connection pool"
   ✅ "Ensuring database schema is up to date..."
   ✅ "Database schema verified."
   ✅ "Server running on http://localhost:8080"

3. Should NOT see:
   ❌ "There was an error establishing an SSL connection"
   ❌ "connect ECONNREFUSED"
   ❌ Any SSL-related errors

4. Test your app:
   • Visit: https://campusmart-7d3d4.web.app
   • Try to register
   • Should work! ✅

═══════════════════════════════════════════════════════════════
  ALTERNATIVE: USE SERVICE REFERENCE
═══════════════════════════════════════════════════════════════

Instead of copying the URL, reference it directly:

1. In campusmart service → Variables

2. Edit DATABASE_URL

3. Use this format:
   ${{Postgres.DATABASE_PRIVATE_URL}}

   Or:
   ${{Postgres.RAILWAY_PRIVATE_DOMAIN}}

4. Railway will automatically use the private network URL

5. This keeps the connection in sync if database changes

═══════════════════════════════════════════════════════════════
  TROUBLESHOOTING
═══════════════════════════════════════════════════════════════

Problem: Can't find "Private Network" section
Solution:
  • Look for "Internal URL" or "Service URL"
  • Check "Connect" or "Settings" tab
  • Look for any URL with ".railway.internal"
  • If not available, your plan might not support it

Problem: Private URL not working
Solution:
  • Make sure both services are in same project
  • Check spelling of URL (must be exact)
  • Try restarting both services
  • Wait 5 minutes for network to propagate

Problem: Still getting connection errors
Solution:
  • Check DATABASE_URL was actually updated
  • Look at logs for exact error message
  • Try public URL with SSL config instead
  • See: RAILWAY_SSL_ALTERNATIVE_FIXES.txt

Problem: "railway.internal" not resolving
Solution:
  • Services must be in same Railway project
  • Private networking might not be enabled
  • Try using service reference: ${{Postgres.DATABASE_URL}}
  • Contact Railway support

═══════════════════════════════════════════════════════════════
  WHY PRIVATE NETWORKING IS BETTER
═══════════════════════════════════════════════════════════════

Public Connection (with SSL):
  Backend → Internet → Railway Edge → SSL Check → Database
  • Slower (more hops)
  • Requires SSL configuration
  • Can have certificate issues
  • Exposed to internet

Private Connection (no SSL needed):
  Backend → Railway Internal Network → Database
  • Faster (direct connection)
  • No SSL configuration needed
  • No certificate issues
  • Not exposed to internet
  • More secure

  ADDITIONAL OPTIMIZATIONS
═══════════════════════════════════════════════════════════════

After setting up private networking, you can also:

1. Remove SSL-related environment variables:
   • PGSSLMODE (not needed anymore)
   • NODE_TLS_REJECT_UNAUTHORIZED (not needed)

2. Update connection pool settings for better performance:
   • Add: DB_POOL_MAX=20
   • Add: DB_POOL_IDLE_TIMEOUT=30000

3. Enable connection pooling:
   • Railway handles this automatically
   • No additional config needed
