═══════════════════════════════════════════════════════════════
  💡 WHY THIS WORKS
═══════════════════════════════════════════════════════════════

Your error "ENOTFOUND host" means DATABASE_URL has the literal
word "host" instead of the actual database hostname.

Using ${{Postgres.DATABASE_URL}} tells Railway to automatically
insert the correct database URL, which includes:
  • Real hostname (containers-us-west-xxx.railway.app)
  • Real password (long random string)
  • Correct port (5432)
  • Database name (railway)

This fixes the hostname error AND keeps it in sync if the
database URL ever changes.

Adding PGSSLMODE=no-verify allows SSL connections without
certificate validation, which Railway PostgreSQL requires.
