  🔧 FIX: "getaddrinfo ENOTFOUND host"
  Database hostname is missing or incorrect
═══════════════════════════════════════════════════════════════

ERROR: "getaddrinfo ENOTFOUND host"
MEANING: Your backend can't find the database because the hostname
         is literally "host" instead of the actual database URL

GOOD NEWS: This is easier to fix than SSL errors!

═══════════════════════════════════════════════════════════════
  🎯 SOLUTION: FIX DATABASE_URL IN RAILWAY
═══════════════════════════════════════════════════════════════

The problem is your DATABASE_URL variable is incorrect or missing.

STEP 1: Get Correct Database URL
────────────────────────────────────────────────────────────────
1. Go to: https://railway.app/dashboard

2. Click your campusmart project

3. Click on "Postgres" service (the database, not campusmart)

4. Click "Connect" tab

5. Look for "DATABASE_URL" or "Connection String"

6. Copy the FULL URL - should look like:
   postgresql://postgres:LONG_PASSWORD@containers-us-west-123.railway.app:5432/railway

   NOT like:
   postgresql://postgres:password@host:5432/railway ❌

7. Make sure it has:
   ✅ Real hostname (ends with .railway.app or .railway.internal)
   ✅ Real password (long random string)
   ✅ Port 5432
   ✅ Database name "railway"

STEP 2: Update DATABASE_URL in Backend Service
────────────────────────────────────────────────────────────────
1. Go back to campusmart service (your backend)

2. Click "Variables" tab

3. Find DATABASE_URL variable

4. Click "Edit" or pencil icon

5. Replace with the correct URL from Step 1

6. Make sure there are NO extra spaces or quotes

7. Click "Save" or "Update"

8. Railway will auto-redeploy (wait 2 minutes)

STEP 3: Verify It's Fixed
────────────────────────────────────────────────────────────────
1. After redeploy, click "View Logs"

2. Look for:
   ✅ "Database: Initialized PostgreSQL connection pool"
   ✅ "Database schema verified"
   ✅ "Server running on http://localhost:8080"

3. Should NOT see:
   ❌ "getaddrinfo ENOTFOUND host"
   ❌ "ENOTFOUND"

4. Test app: https://campusmart-7d3d4.web.app

═══════════════════════════════════════════════════════════════
  🔍 ALTERNATIVE: USE SERVICE REFERENCE
═══════════════════════════════════════════════════════════════

Instead of copying the URL, reference it directly:

1. In campusmart service → Variables

2. Edit DATABASE_URL

3. Replace value with:
   ${{Postgres.DATABASE_URL}}

4. This automatically uses the correct Postgres URL

5. Railway keeps it in sync if database changes

6. Save and wait for redeploy

═══════════════════════════════════════════════════════════════
  🔍 CHECK IF DATABASE_URL EXISTS
═══════════════════════════════════════════════════════════════

If DATABASE_URL variable doesn't exist:

1. In campusmart service → Variables

2. Click "New Variable"

3. Add:
   Name: DATABASE_URL
   Value: (paste URL from Postgres service)

4. Or use reference:
   Name: DATABASE_URL
   Value: ${{Postgres.DATABASE_URL}}

5. Save and wait for redeploy

═══════════════════════════════════════════════════════════════
  🔍 VERIFY POSTGRES SERVICE IS RUNNING
═══════════════════════════════════════════════════════════════

Make sure your database is actually running:

1. Go to Railway dashboard

2. Click Postgres service

3. Check status - should show "Active" or "Running"

4. If not running:
   • Click "..." menu
   • Click "Restart"
   • Wait 1 minute

5. If still not running:
   • Check Railway status: https://railway.app/status
   • May need to recreate database

═══════════════════════════════════════════════════════════════
  🔍 COMMON CAUSES OF THIS ERROR
═══════════════════════════════════════════════════════════════

1. DATABASE_URL has placeholder "host" instead of real hostname
   Fix: Copy correct URL from Postgres service

2. DATABASE_URL variable doesn't exist
   Fix: Add it with correct value

3. DATABASE_URL has wrong format
   Fix: Should be postgresql://user:pass@host:port/db

4. Postgres service not in same project
   Fix: Both services must be in same Railway project

5. DATABASE_URL has extra spaces or quotes
   Fix: Remove any extra characters

═══════════════════════════════════════════════════════════════
  📋 CORRECT DATABASE_URL FORMAT
═══════════════════════════════════════════════════════════════

CORRECT FORMAT:
postgresql://postgres:abc123xyz@containers-us-west-123.railway.app:5432/railway

PARTS:
  • Protocol: postgresql://
  • Username: postgres
  • Password: abc123xyz (long random string)
  • Host: containers-us-west-123.railway.app (real Railway host)
  • Port: 5432
  • Database: railway

WRONG FORMATS:
❌ postgresql://postgres:password@host:5432/railway
   (has "host" instead of real hostname)

❌ postgresql://postgres:password@localhost:5432/railway
   (has localhost instead of Railway host)

❌ "postgresql://postgres:password@host:5432/railway"
   (has quotes around it)

❌ postgresql://postgres:password@host:5432/railway 
   (has extra space at end)
